3. RUNNING THE APPLICATION (Local Setup)
--------------------------------------------------------------------------------

  OPTION A — Use the deployed version (No setup required)
  -----------------------------------------------
    Simply visit: https://jotify-production.up.railway.app
    Use the test accounts above to log in immediately.

  OPTION B — Run locally
  -----------------------------------------------

  Requirements:
    - PHP 8.2+, Composer, Node.js 18+, npm, MySQL 8.0+

  Step 1. Install dependencies:
    composer install
    npm install

  Step 2. Set up environment:
    cp .env.example .env          (Linux/macOS)
    copy .env.example .env        (Windows)
    php artisan key:generate

  Step 3. Configure .env (minimum required values):
    APP_URL=http://localhost:8000
    DB_DATABASE=jotify
    DB_USERNAME=root
    DB_PASSWORD=

  Step 4. Set up database:
    php artisan migrate
    php artisan db:seed

    Or import the SQL dump directly:
      mysql -u root -p jotify < database/jotify_database.sql

  Step 5. Link storage (required for images and avatars):
    php artisan storage:link

  Step 6. Build frontend assets:
    npm run build

  Step 7. Start the server:
    php artisan serve

    Access at: http://localhost:8000

6. REALTIME COLLABORATION
--------------------------------------------------------------------------------

  Real-time collaboration is implemented using Laravel Echo + Pusher WebSocket.

  How it works:
    - When a note is shared with "edit" permission, all collaborators can
      type simultaneously and see each other's changes live.
    - The deployed version at Railway has Pusher fully configured.

  To enable realtime locally:
    1. Create a free account at https://pusher.com
    2. Create a Channels app (select cluster ap1)
    3. Add the App ID, Key, Secret, Cluster to .env
    4. Rebuild assets: npm run build
    5. Restart: php artisan serve

  Without Pusher configured locally:
    All other features work normally. Realtime sync is simply disabled.
    No errors occur.

8. OPTIONAL FEATURES (Section 2.3 — Advanced Note Management)
--------------------------------------------------------------------------------

  The following optional features from Section 2.3 are fully implemented:

  [OPT-1] PASSWORD-PROTECTED NOTES
  ---
  - Users can lock any note with a unique per-note password.
  - Once enabled, a password dialog appears before viewing, editing,
    or deleting the note.
  - Users can change the note password or disable protection entirely.
  - Each note has its own independent password, unrelated to others.
  - Implemented in: NoteController (setPassword, unlockNote,
    changeNotePassword, removePassword methods).

  [OPT-2] NOTE SHARING WITH PERMISSION CONTROL
  ---
  - Note owners can share notes with other registered users by email.
  - Sharing supports two permission levels: read-only and edit.
  - Multiple recipients can be added to the same note.
  - The owner can revoke access or modify permissions at any time
    from the share management panel inside the editor.
  - Implemented in: ShareController (share, getShares,
    updatePermission, revoke methods).

  [OPT-3] DEDICATED "SHARED WITH ME" SECTION
  ---
  - Recipients have a dedicated page (/shared) listing all notes
    shared with them.
  - Each entry clearly shows:
      * The note title
      * Sharing status (read-only or editable)
      * Who shared the note (sharer's name)
      * When the note was shared (timestamp)
  - Implemented in: ShareController::sharedWithMe(),
    resources/views/notes/shared.blade.php

  [OPT-4] REAL-TIME COLLABORATION VIA WEBSOCKET
  ---
  - For notes shared with "edit" permission, multiple users can
    edit the note simultaneously.
  - Changes made by one user are broadcast live to all other editors
    via Pusher WebSocket (Laravel Echo + Laravel Broadcasting).
  - All collaborators observe each other's changes in real time
    without refreshing the page.
  - Implemented in: NoteContentUpdated event, ShareController::
    autoSaveShared(), NoteController::autoSave(), routes/channels.php

  [OPT-5] STATUS ICONS FOR SPECIAL NOTES
  ---
  - Notes that are pinned, shared, or password-protected display
    recognizable status icons on their note cards.
  - Icons are visible in BOTH grid view and list view layouts.
  - This allows evaluators (and users) to immediately identify a
    note's status without opening it.
  - Icons used:
      * 📌 (push_pin)   — Pinned note
      * 🔗 (people)      — Shared note
      * 🔒 (lock)        — Password-protected note
  - Implemented in: resources/views/notes/partials/note-card.blade.php

9. TROUBLESHOOTING
--------------------------------------------------------------------------------

  Images or avatars not showing locally:
    → Run: php artisan storage:link

  500 error or blank page after setup:
    → Run: php artisan config:clear
            php artisan view:clear
            php artisan cache:clear

  Migration fails (database not found):
    → Create the database manually first:
      CREATE DATABASE jotify CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

  composer install fails (PHP version error):
    → Ensure PHP 8.2+ is active. On XAMPP, use the XAMPP shell.

  npm run build fails:
    → Ensure Node.js 18+ is installed. Run npm install first.

  Application key missing error:
    → Run: php artisan key:generate
